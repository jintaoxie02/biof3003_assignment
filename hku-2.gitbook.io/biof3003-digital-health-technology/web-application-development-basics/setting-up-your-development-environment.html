Setting Up Your Development Environment

Now that you’re familiar with the components of the HeartLens app, let’s set up your development environment. By the end of this session, you’ll have a fully functional Next.js project ready to build upon.

1. Install and Set Up Node.js

Step 1: Download and Install Node.js

  1. Go to the official Node.js website .

  2. Download the LTS (Long-Term Support) version. This version is stable and recommended for most users.

  3. Run the installer and follow the prompts to complete the installation.

Step 2: Verify the Installation

After installation, open your terminal or command prompt and run the following commands to verify that Node.js and npm (Node Package Manager) are installed correctly:

node -v
npm -v

You should see the versions of Node.js and npm printed in the terminal. For example:

v18.17.1
9.6.7

Step 3:Set Up Node.js in VSCode

  • Open VSCode and ensure it’s configured to use the terminal. You can open the integrated terminal by pressing `Ctrl + `` (backtick) or navigating to View > Terminal .

  • You’ll use this terminal to run commands for setting up and running your Next.js app.

2. Create a Next.js Application

Step 1: Use create-next-app to Generate the Project

Next.js provides a convenient command-line tool called create-next-app to scaffold a new project. Follow these steps:

  1. Open your terminal in VSCode.

  2. Run the following command to create a new Next.js app:

  3. When prompted, select the following default options:

    • Would you like to use TypeScript? → Yes

    • Would you like to use ESLint? → Yes

    • Would you like to use Tailwind CSS? → Yes

    • Would you like to use src/ directory? → No

    • Would you like to use App Router? → Yes

    • Would you like to use Turbopack for `next dev`? → No

    • Would you like to customize the default import alias? → No

    These options will set up a modern Next.js project with TypeScript, Tailwind CSS, and the App Router.

  4. Once the setup is complete, navigate into your project folder:

cd heartlens

3. Explore the Project Structure

After creating the project, let’s take a quick look at the folder structure and key files:

Key Folders and Files

  • app/ : Contains the pages and layouts for your app. The App Router is used here, which is the newest routing system in Next.js.

  • public/ : Stores static assets like images, fonts, or other files that need to be served directly.

  • package.json : Lists all dependencies and scripts for your project.

  • tailwind.config.js : Configuration file for Tailwind CSS.

  • tsconfig.json : Configuration file for TypeScript.

Technologies Used

  • TypeScript : A typed superset of JavaScript that improves code quality and reduces errors.

  • Tailwind CSS : A utility-first CSS framework for quickly styling your app.

  • App Router : The newest routing system in Next.js, which simplifies navigation and layout management.

4. Run the Next.js App

Step 1: Start the Development Server

To run your app, use the following command in the terminal:

npm install
npm run dev

Open your browser and go to http://localhost:3000 to view the app. You will see this appears.

What's Next

Now that your development environment is set up and you’ve successfully created a Next.js app, it’s time to dive into building the core functionality of the HeartLens app. In the next step, we’ll walk through how to create the Heart Rate and Heart Rate Variability (HRV) components using React’s useState and useEffect hooks. These hooks are essential for managing state (like heart rate values) and handling side effects (like simulating real-time data updates).

For now, we’ll focus on the logic and structure of these components, keeping the styling simple to help you grasp the fundamentals of React. Once you’re comfortable with the basics of React and component communication, we’ll introduce Tailwind CSS to style the app and make it visually appealing. Tailwind CSS will be covered in a later session, as it’s best introduced after you’ve built a solid understanding of React’s core concepts.
