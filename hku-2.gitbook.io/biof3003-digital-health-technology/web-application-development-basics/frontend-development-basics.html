Frontend Development Basics

We’ll start with the simplest components and gradually move toward more complex ones. To make the most of our time, we’ll alternate between instruction and hands-on coding exercises, giving you opportunities to practice what you’ve learned.

1. Start with the SignalConfig Component

Radio buttons allow users to explicitly choose one option from a predefined list. This approach is more intuitive than cycling through options with a toggle button, especially when there are multiple modes to choose from.

Step 1: Create the SignalConfig Component

Add a new file called SignalConfig.tsx in the app/components/ directory:

"use client";
import { useState } from "react";

export default function SignalConfig() {
  // State to manage the selected mode
  const [mode, setMode] = useState("Green Only");

  // State to toggle visibility of the radio buttons
  const [isConfigOpen, setIsConfigOpen] = useState(false);

  return (
    <div>
      {/* Header */}
      <h3>Signal Configuration</h3>

      {/* Display Current Mode */}
      <p>Selected Mode: {mode}</p>

      {/* Toggle Button */}
      <button
        onClick={() => setIsConfigOpen(!isConfigOpen)} // Toggle visibility of radio buttons
      >
        {isConfigOpen ? "Hide Options" : "Configure Signal"}
      </button>

      {/* Radio Buttons (Conditionally Rendered) */}
      {isConfigOpen && (
        <div>
          <label>
            <input
              type="radio"
              name="signal-mode"
              value="Green Only"
              checked={mode === "Green Only"}
              onChange={(e) => setMode(e.target.value)}
            />
            Green Only
          </label>
          <br />
          <label>
            <input
              type="radio"
              name="signal-mode"
              value="Red Only"
              checked={mode === "Red Only"}
              onChange={(e) => setMode(e.target.value)}
            />
            Red Only
          </label>
          <br />
          <label>
            <input
              type="radio"
              name="signal-mode"
              value="Blue Only"
              checked={mode === "Blue Only"}
              onChange={(e) => setMode(e.target.value)}
            />
            Blue Only
          </label>
          <br />
          <label>
            <input
              type="radio"
              name="signal-mode"
              value="2R - G - B"
              checked={mode === "2R - G - B"}
              onChange={(e) => setMode(e.target.value)}
            />
            2R - G - B
          </label>
          <br />
          <label>
            <input
              type="radio"
              name="signal-mode"
              value="Custom"
              checked={mode === "Custom"}
              onChange={(e) => setMode(e.target.value)}
            />
            Custom
          </label>
        </div>
      )}
    </div>
  );
}

Explanation of the Code

  1. useState Hook :

    • mode: Stores the currently selected signal mode (default is "Green Only").

    • isConfigOpen: Tracks whether the radio buttons are visible (true) or hidden (false).

  2. Toggle Button :

    • The button toggles the isConfigOpen state between true and false.

    • The label changes dynamically based on the isConfigOpen state:

      • Shows "Configure Signal" when closed.

      • Shows "Hide Options" when open.

  3. Radio Buttons :

    • Each radio button represents a signal mode (e.g., "Green Only", "Red Only").

    • The checked attribute ensures that the correct radio button is marked as selected based on the current mode.

    • The onChange event updates the mode state when a user selects a different option.

  4. Conditional Rendering :

    • The {isConfigOpen && (...)} syntax ensures the radio buttons are only displayed when isConfigOpen is true.

Step 2: Integrate the Component

Update page.tsx to include the SignalConfig component:

import SignalConfig from "./components/SignalConfig";
import HeartRateMonitor from "./components/HeartRateMonitor";
export default function Home() {
  return (
    <div>
      <h1>HeartLens App</h1>
      <HeartRateMonitor />
      <SignalConfig />
    </div>
  );
}

2. Build the Camera Feed

The Camera Feed simulates the live camera input used to collect PPG data. For now, we’ll use a placeholder video element to represent the camera feed.

Step 1: Add a Placeholder Camera Feed

Create a new file called CameraFeed.tsx:

"use client";
import { useState } from "react";

export default function CameraFeed() {
  const [stream, setStream] = useState<MediaStream | null>(null);
  const [hasPermission, setHasPermission] = useState(false);

  // Function to start the camera
  const startCamera = async () => {
    try {
      // Check if we're in a secure context (HTTPS required for camera access)
      if (!window.isSecureContext) {
        console.error("Camera access requires HTTPS");
        return;
      }

      // Request camera access
      const newStream = await navigator.mediaDevices.getUserMedia({
        video: { facingMode: "environment" }, // Use the rear camera
      });

      // Set permission state
      setHasPermission(true);

      // Display the video stream in the <video> element
      const videoElement = document.getElementById("camera-feed") as HTMLVideoElement;
      if (videoElement) {
        videoElement.srcObject = newStream;
        videoElement.play();
      }

      // Save the stream for cleanup later
      setStream(newStream);
    } catch (err) {
      console.error("Error accessing camera:", err);
      setHasPermission(false);
    }
  };

  // Function to stop the camera
  const stopCamera = () => {
    if (stream) {
      stream.getTracks().forEach((track) => track.stop());
      setStream(null);
      setHasPermission(false);
    }
  };

  return (
    <div>
      <h3>Camera Feed</h3>
      <button onClick={startCamera} disabled={hasPermission}>
        Start Camera
      </button>
      <button onClick={stopCamera} disabled={!hasPermission}>
        Stop Camera
      </button>
      <br />
      <video id="camera-feed" width="320" height="240" autoPlay playsInline></video>
    </div>
  );
}

Explanation of the Code

  1. State Management :

    • stream: Stores the camera stream so it can be stopped later.

    • hasPermission: Tracks whether the user has granted camera access.

  2. startCamera Function :

    • Uses navigator.mediaDevices.getUserMedia to request access to the camera.

    • The await keyword ensures the function waits for the user’s permission before proceeding.

    • Once access is granted, the video stream is displayed in the <video> element.

  3. stopCamera Function :

    • Stops all active tracks in the camera stream and resets the state.

  4. UI Elements :

    • Two buttons allow users to start and stop the camera.

    • The <video> element displays the live camera feed.

Update page.tsx to include the CameraFeed component:

3. Build and Integrate the Real-Time Visualization Panel

The Real-Time Visualization Panel displays the PPG signal as a line graph. For now, we’ll use simulated data to create a smooth learning experience.

Step 1: Install Chart.js

Install Chart.js , a popular library for creating charts in React:

npm install chart.js react-chartjs-2

Step 2: Create the VisualizationPanel Component

Create a new file called VisualizationPanel.tsx:

"use client";
import { useEffect, useState } from "react";
import { Line } from "react-chartjs-2";

export default function VisualizationPanel() {
  const [ppgData, setPpgData] = useState<number[]>([]);

  // Simulate real-time PPG data
  useEffect(() => {
    const interval = setInterval(() => {
      const newData = Math.random() * 100; // Simulated PPG value
      setPpgData((prevData) => [...prevData.slice(-20), newData]); // Keep last 20 points
    }, 500);

    return () => clearInterval(interval); // Cleanup timer when component unmounts
  }, []);

  // Chart.js configuration
  const chartData = {
    labels: Array.from({ length: ppgData.length }, (_, i) => i.toString()),
    datasets: [
      {
        label: "PPG Signal",
        data: ppgData,
        borderColor: "rgb(75, 192, 192)",
        tension: 0.4,
        fill: true,
        backgroundColor: "rgba(75, 192, 192, 0.2)",
        pointRadius: 0,
      },
    ],
  };

  const chartOptions = {
    responsive: true,
    scales: {
      y: {
        beginAtZero: false,
      },
    },
    animation: {
      duration: 0, // Disable animation for better performance
    },
  };

  return (
    <div>
      <h3>Real-Time Visualization</h3>
      <Line data={chartData} options={chartOptions} />
    </div>
  );
}

Explanation of the Code

  1. Simulated Data :

    • A timer (setInterval) generates random PPG values every 500ms.

    • The ppgData state stores the last 20 values for visualization.

  2. Chart.js Configuration :

    • chartData: Defines the data and appearance of the line graph.

      • labels: Represents the x-axis (time steps).

      • datasets: Contains the PPG signal data.

    • chartOptions: Configures the chart’s behavior (e.g., responsiveness, animations).

  3. Dynamic Updates :

    • The Line component from react-chartjs-2 renders the line graph.

    • As new data is added to ppgData, the chart updates automatically.

Step 3: Integrate the Component

Update page.tsx to include the VisualizationPanel component:

import SignalConfig from "./components/SignalConfig";

import VisualizationPanel from "./components/VisualizationPanel";

export default function Home() {
  return (
    <div>
      <h1>HeartLens App</h1>
      <CameraFeed />
      <VisualizationPanel />
      <HeartRateMonitor />
      <SignalConfig />
    </div>
  );
}
