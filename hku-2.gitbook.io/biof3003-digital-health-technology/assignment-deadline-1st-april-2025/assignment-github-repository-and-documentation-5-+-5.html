  1. Project Overview

Briefly describe the purpose of the app.

The HeartLen App is a web-based tool designed to process photoplethysmography (PPG) signals captured via a webcam. It calculates heart rate, heart rate variability (HRV), and signal quality using machine learning models. The processed data can be saved to a MongoDB database for further analysis.

Part 2: Documentation

Objective

Write a Word document that explains the functionality of the app in detail. The document should cover the following topics:

1. Function of Each Component

Explain the role of each major component in the app:

  • CameraFeed :

    • Captures video frames from the webcam and displays them on a canvas element.

    • Used to extract RGB values for PPG signal processing.

  • ChartComponent :

    • Visualizes the PPG signal and detected valleys in real-time.

  • MetricsCard :

    • Displays key metrics such as heart rate, HRV, and signal quality.

  • SignalCombinationSelector :

    • Allows users to select different signal combinations (e.g., red-only, green-only).

2. Function of Each State Variable

Describe the purpose of each state variable in the main page.tsx file:

  • isRecording :

    • Tracks whether the app is currently recording video frames.

  • signalCombination :

    • Stores the selected signal combination (e.g., "redOnly", "greenOnly").

  • ppgData :

    • Holds the processed PPG signal data.

  • valleys :

    • Stores the indices and values of detected valleys in the PPG signal.

  • heartRate :

    • Stores the calculated heart rate and its confidence level.

  • hrv :

    • Stores the calculated HRV and its confidence level.

  • isUploading :

    • Tracks whether data is being uploaded to MongoDB.

3. Data Processing Pipeline

Explain how data flows from the frontend to the backend:

  1. Frontend :

    • The usePPGProcessing hook processes video frames to extract PPG signals.

    • Features such as heart rate, valleys, and HRV are calculated in real-time.

    • Signal quality is assessed using a TensorFlow.js model.

  2. Backend :

    • The /api/save-record route receives PPG data from the frontend.

    • Data is saved to MongoDB using the RecordSchema.

4. Integration of ML Models

Describe how the TensorFlow.js model is integrated into the app:

  • The useSignalQuality hook loads the TensorFlow.js model from the /public/tfjs_model folder.

  • Features are extracted from the PPG signal and passed to the model for classification.

  • The model predicts the signal quality (e.g., "Good", "Acceptable", "Bad") and returns a confidence score.
