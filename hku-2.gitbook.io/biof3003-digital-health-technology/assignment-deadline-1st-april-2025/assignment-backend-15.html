Assignment: Backend (15%)

In this assignment, you will work on training a TensorFlow model to classify PPG signals into three categories: Good , Acceptable , and Bad . The goal is to ensure that you understand the end-to-end process of data labeling, feature extraction, model training, and integration into the app. Below are the key requirements:

1. Manual Labeling of PPG Data

  • Use the provided MongoDB URI to retrieve the collected PPG data.

  • Visualize the PPG signals using the plot_ppg_signals function (covered in the previous session).

  • Manually label the signals into three categories:

    • Good : High-quality signals suitable for analysis.

    • Acceptable : Moderate-quality signals that can still provide useful insights.

    • Bad : Low-quality or noisy signals that are unreliable.

  • Save the indices of the labeled signals into three lists: excellent_samples, acceptable_samples, and bad_samples

2. Feature Extraction

  • Customize the feature extraction logic in the useSignalQuality hook.

  • Modify the default feature extraction function to include additional or alternative features such as:

    • Frequency-domain features (e.g., FFT coefficients).

    • Statistical measures (e.g., median, variance).

    • Domain-specific features relevant to PPG signal quality.

  • Ensure that your custom feature extraction logic is reflected in both the frontend (useSignalQuality) and backend (training script).

3. Training the TensorFlow Model

  • rain a TensorFlow model using the labeled data and extracted features.

  • Use the following steps as a guide:

    1. Prepare the dataset by extracting features from the PPG signals.

    2. Split the dataset into training and testing sets.

    3. Define and train a classification model using TensorFlow.

    4. Evaluate the model's performance and save it for deployment.

  • Important Note : Using the default model will result in a mark deduction. You are required to customize both the feature extraction and model architecture.

4. Incorporating the Trained Model

  • Convert the trained TensorFlow model into TensorFlow.js format for use in the frontend.

  • Place the converted model in the public/model folder of your app.

  • Update the useSignalQuality hook to load and use the TensorFlow.js model for real-time signal quality classification

Submission Guideline

  1. Submit your Jupyter Notebook in HTML format on Moodle

  2. Ensure the trained model is included in the public/model folder.
