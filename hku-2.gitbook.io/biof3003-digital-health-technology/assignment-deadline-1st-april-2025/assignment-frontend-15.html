In this assignment, you will have enhanced the app with:

  1. A User Panel for inputting and confirming subject IDs.

  2. Subject-specific historical data retrieval (heart rate, HRV, and last access date).

  3. Subject-specific data storage in MongoDB.

  4. A redesigned layout using Tailwind CSS .

Let's dive into each feature step-by-step.

1 User Panel: Input and Confirm Subject ID

Objective

Add a user panel where users can:

  • Input their subjectId.

  • Click a "Confirm User" button to confirm their identity.

  • Once confirmed, fetch their historical data and last access date.

Steps

1.1 Add Input Field and Button

  • In app/page.tsx, create an input field for the subjectId and a "Confirm User" button.

  • Use React state (useState) to store the currentSubject and confirmedSubject.

Scaffold Code :

const [currentSubject, setCurrentSubject] = useState('');
const [confirmedSubject, setConfirmedSubject] = useState('');

// Confirm User Function
const confirmUser = () => {
  if (currentSubject.trim()) {
    setConfirmedSubject(currentSubject.trim());
  } else {
    alert('Please enter a valid Subject ID.');
  }
};

return (
  <div>
    {/* Input Field */}
    <input
      type="text"
      value={currentSubject}
      onChange={(e) => setCurrentSubject(e.target.value)}
      placeholder="Enter Subject ID"
      className="border border-gray-300 rounded-md p-2"
    />
    {/* Confirm Button */}
    <button
      onClick={confirmUser}
      className="bg-cyan-500 text-white px-4 py-2 rounded-md ml-2"
    >
      Confirm User
    </button>
  </div>
);

Objective

  • Average heart rate (avgHeartRate).

  • Average HRV (avgHRV).

  • Last access date (lastAccess).

Steps

2.1 Update MongoDB Schema

Modify the MongoDB schema to include subjectId in each record. For example:

2.2 Create a New API Route

Create a new API route (/api/last-access) to fetch the last access date for a specific subjectId.

Scaffold Code :

export async function GET(request) {

    return NextResponse.json({ success: false, error: 'Missing subjectId' });

  try {

    if (!lastRecord) {

    return NextResponse.json({ success: false, error: error.message });
  }
}

2.3 Display Historical Data

Modify the API Route (/api/save-record) by fetch the subect-specific average historical records

Scaffold Code: 

export async function GET(request: Request) {
  try {
    await dbConnect();

    // Extract subjectId from query parameters
    const url = new URL(request.url);
    const subjectId = url.searchParams.get('subjectId');

    if (!subjectId) {
      return NextResponse.json(
        { success: false, error: 'Missing subjectId in query parameters' },
        { status: 400 }
      );
    }

    // Aggregate historical data using MongoDB aggregation pipeline
    const pipeline = [
      {
        $match: { subjectId }, // Filter records by subjectId
      },
      {
        $group: {
          _id: null,
          avgHeartRate: { $avg: '$heartRate.bpm' },
          avgHRV: { $avg: '$hrv.sdnn' },
        },
      },
    ];

    const result = await Record.aggregate(pipeline);

    if (!result.length) {
      // Return default values if no data exists for the subjectId
      return NextResponse.json(
        { success: true, avgHeartRate: 0, avgHRV: 0 },
        { status: 200 }
      );
    }

    const { avgHeartRate, avgHRV } = result[0];
    return NextResponse.json(
      { success: true, avgHeartRate, avgHRV },
      { status: 200 }
    );
  } catch (error: any) {
    return NextResponse.json(
      { success: false, error: error.message },
      { status: 500 }
    );
  }
}

4. Redesign Layout Using Tailwind CSS

Objective

Redesign the app layout to improve usability and aesthetics. You are free to experiment with different layouts as long as it differs from the current structure. Use Tailwind CSS to style components creatively.

Steps

4.1 Organize Components

  • Use a flexbox or grid layout to organize the components:

    • Left Column: Camera Feed .

    • Right Column: Chart , Metrics Cards , and User Panel .

4.2 Example Layout Ideas

Here are some ideas to inspire your redesign:

  1. Sidebar Layout :

    • Place the User Panel and Metrics Cards in a collapsible sidebar.

    • Use the main area for the Camera Feed and Chart .

  2. Stacked Layout :

    • Stack the Camera Feed , Chart , and Metrics Cards vertically for mobile-friendly designs.

  3. Dashboard Layout :

    • Use a grid system to display all components side-by-side in a dashboard-like interface.
