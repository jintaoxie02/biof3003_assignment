App Deployment

Now that you've built and tested your HeartLen application locally, it's time to deploy it so others can access it online. Deployment involves several steps, from ensuring your app builds correctly to pushing your code to GitHub and configuring your deployment environment on Vercel. Letâ€™s go through each step in detail.

Step 1: Build Your App Locally

Before deploying, it's crucial to ensure your app can be built successfully on your local machine. This step helps catch any syntax errors or issues with your code.

  1. Open your terminal and navigate to your project directory:

cd path/to/your/project

  1. Run the following command to check for syntax errors and other issues:

npm run lint

  • What this does: The npm run lint command scans your code for potential syntax errors, formatting issues, and best practices violations. It ensures your code adheres to standard conventions.

  • Why it's important: Fixing linting errors before building prevents runtime issues and ensures smoother deployment.

  1. After fixing any linting issues, build your app using:

npm run build

  • What this does: The npm run build command compiles your React app into optimized static files (HTML, CSS, JavaScript) ready for production.

  • Why it's important: If the build fails, your app won't work when deployed. Fix any errors reported during this step.

  1. Once the build succeeds, test your app locally by running:

npm run start

  • What this does: Starts a local development server to verify that your app works as expected after the build.

  • Why it's important: Ensures that your app behaves correctly before deploying it.

Step 2: Push Your Code to GitHub

If you initially cloned the starter repository from your instructor, you need to update the remote origin to point to your own GitHub repository. This allows you to push your changes to your personal repository.

Instructions

  1. Create a new repository on GitHub:

    • Go to GitHub and click the "+" icon in the top-right corner, then select "New repository."

    • Name your repository (e.g., heartlen-app) and choose whether to make it public or private.

    • Do not initialize the repository with a README, .gitignore, or license file.

  2. Update the remote origin URL in your local repository:

git remote set-url origin https://github.com/your-username/your-repo-name.git

Replace your-username and your-repo-name with your GitHub username and repository name.

  • What this does: Changes the remote URL of your local repository to point to your new GitHub repository instead of the original starter repository.

  • Why it's important: Ensures that your changes are pushed to your personal repository, not the instructor's.

  1. Verify the updated remote URL:

git remote -v

You should see your new repository URL listed under origin.

  1. Commit and push your changes:

git add .
git commit -m "Initial commit with completed HeartLen app"
git push origin main

  • What this does: Stages all your changes, commits them with a message, and pushes them to the main branch of your GitHub repository.

  • Why it's important: Pushing your code to GitHub makes it accessible for deployment and version control.

Step 3: Deply to Vercel

Vercel is a popular platform for deploying React applications. It provides a seamless process for hosting your app and managing environment variables.

Instructions

  1. Sign up for a free Vercel account:

    • Go to Vercel and sign up using your GitHub account.

  2. Import your GitHub repository:

    • In the Vercel dashboard, click "Add New" and select "Project."

    • Choose "Import Project from GitHub" and authorize Vercel to access your repositories.

    • Select the repository you just pushed (e.g., heartlen-app).

  3. Configure the deployment settings:

    • Environment Variables: Add the required environment variables for your app (e.g., MONGODB_URI).

      • Click "Environment Variables" and add the key-value pairs needed for your app to connect to MongoDB.

      • Example:

        • Key: MONGODB_URI

        • Value: mongodb+srv://<username>:<password>@cluster0.nmjaz.mongodb.net/?retryWrites=true&w=majority

      • What this does: Environment variables store sensitive information like database credentials securely, without exposing them in your code.

      • Why it's important: Protects your app from unauthorized access and keeps sensitive data secure.

    • Build and Deployment: Change the install command into npm install --legacy-peer-deps to bypass peer dependency conflicts

  4. Deploy your app:

    • Click "Deploy" to start the deployment process.

    • Vercel will automatically build and deploy your app. Once complete, you'll receive a live URL (e.g., https://heartlen-app.vercel.app)

  5. Test your deployed app:

    • Open the live URL provided by Vercel and verify that your app works as expected.

    • Check that features like recording, sampling, and saving data to MongoDB function correctly.
