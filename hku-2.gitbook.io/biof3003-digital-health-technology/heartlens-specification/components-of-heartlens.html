1. User Interface

The User Interface (UI) is the primary way users interact with the app. It includes several buttons and controls that allow users to manage data collection, configuration, and analysis.

a. Start Recording Button

  • Purpose : Initializes the camera feed and begins collecting PPG data.

  • Functionality : When pressed, this button activates the phoneâ€™s camera and starts capturing light intensity changes caused by blood flow in the fingertip. It also triggers the real-time processing of PPG signals.

b. Signal Configuration Button

  • Purpose : Allows users to select the mode of extracting PPG data by choosing specific combinations of RGB values obtained from the camera feed. These combinations are designed to optimize the extraction of the PPG signal based on different mathematical transformations of the RGB channels.

  • Functionality :

    • The human fingertip reflects light differently depending on blood flow, and the camera captures this reflected light in three primary color channels: Red (R) , Green (G) , and Blue (B) .

    • Each channel contains unique information about the PPG signal, and certain mathematical combinations of these channels may yield better results for heart rate and HRV estimation.

    • This button lets users toggle between predefined signal combination modes or define a custom combination. The available modes are:

      • 2R - G - B : A weighted combination that emphasizes the red channel while subtracting contributions from green and blue. This mode is useful for scenarios where red light provides stronger signals.

      • Red Only : Focuses solely on the red channel. This mode is often used in low-light conditions or for users with darker skin tones.

      • Green Only : Focuses solely on the green channel. The green channel is highly sensitive to changes in blood volume and is commonly used as the default mode.

      • Blue Only : Focuses solely on the blue channel. While less commonly used, this mode can provide insights in specific lighting conditions.

      • Red - Blue : Subtracts the blue channel from the red channel. This mode can help isolate specific aspects of the PPG signal in certain environments.

      • Custom : Allows users to specify their own mathematical combination of the RGB channels (e.g., aR + bG + cB, where a, b, and c are user-defined coefficients).

    • By experimenting with these modes, users can determine which combination works best for their specific environment, finger placement, and lighting conditions.

c. Send Button

  • Purpose : Sends the collected PPG data to a backend or processing module for further analysis.

  • Functionality : Once data collection is complete, users can press this button to send the PPG data for advanced processing, such as calculating heart rate, HRV, and confidence levels. The processed results are then displayed on the app.

2. Camera Feed

The Camera Feed is a critical component of the app, as it captures the PPG data used to estimate heart rate and HRV.

  • Purpose : Displays the live camera feed and ensures proper finger placement for accurate PPG signal extraction.

  • Functionality : When the user places their finger over the camera lens, the app detects changes in light intensity caused by blood volume fluctuations. These changes are visualized in real time, providing feedback to ensure optimal signal quality.

  • Visualization : A small preview window shows the camera feed, helping users align their finger correctly.

3. Heart Rate and Heart Rate Variability (HRV) Display

This block provides a clear and concise display of the userâ€™s heart rate, heart rate variability and its confidence level.

  • Purpose : Shows the calculated heart rate in beats per minute (BPM) and indicates the reliability of the measurement.

  • Details :

    • Heart Rate (BPM) : Displays the current heart rate based on the processed PPG data.

    • Heart Rate Variability (HRV): Displays the calculated HRV value.

    • Confidence Level : Indicates how reliable the heart rate measurement is (e.g., high, medium, low). This helps users assess the quality of the data.

4. Real-Time Visualization Panel

The Real-Time Visualization Panel provides a dynamic and interactive way to view the PPG data as itâ€™s being collected.

  • Purpose : Displays the raw and processed PPG signals in real time, allowing users to monitor signal quality and trends.

  • Features :

    • Raw PPG Signal : Shows the unprocessed light intensity changes captured by the camera.

    • Filtered PPG Signal : Displays the processed PPG signal after applying filters to remove noise.

    • Heartbeat Peaks : Highlights detected peaks in the PPG signal, which correspond to individual heartbeats.

  • Visualization : A graph or waveform plot updates dynamically as new data is collected, giving users immediate feedback on their heart activity.

5. How the Components Work Together

These components are interconnected to create a cohesive user experience:

  1. The Start Recording Button initializes the camera feed and begins data collection.

  2. The Signal Configuration Button allows users to adjust settings for optimal signal quality.

  3. The Camera Feed ensures proper finger placement and captures PPG data.

  4. The Heart Rate and HRV Displays show the processed results in an easy-to-understand format.

  5. The Send Button sends the data for further analysis, while the Real-Time Visualization Panel provides live feedback on the PPG signal.

By combining these components, HeartLens delivers a powerful yet user-friendly tool for monitoring cardiovascular health.
