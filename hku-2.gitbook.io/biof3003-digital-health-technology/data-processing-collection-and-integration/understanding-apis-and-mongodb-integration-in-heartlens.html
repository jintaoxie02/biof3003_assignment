Understanding APIs and MongoDB Integration in HeartLens

By the end of this session, students will:

  1. Understand how APIs work in the context of the HeartLen app.

  2. Learn about different types of HTTP requests (GET, POST, PUT, DELETE) and their use cases.

  3. Set up a MongoDB database and configure it for use with the HeartLen app

  4. Explore how data is sent to MongoDB using the pushToMongoDB function in app/page.tsx.

1. Introduction to APIs in Next.js

What is an API?

An API (Application Programming Interface) allows two systems to communicate with each other. In the HeartLen app, we use an API to send PPG data from the frontend to the backend, which then stores it in MongoDB.

API Route in Next.js

In Next.js, API routes are defined in the app/api directory. For example:

  • The file app/api/save-record/route.ts defines an API endpoint (/api/save-record) that handles POST requests.

// app/api/save-record/route.ts
import { NextResponse } from 'next/server';
import mongoose from 'mongoose';

export async function POST(request: Request) {
  try {
    await dbConnect(); // Connect to MongoDB
    const body = await request.json(); // Parse incoming JSON data

    // Create a new record in MongoDB
    const newRecord = await Record.create({
      heartRate: body.heartRate,
      hrv: body.hrv,
      ppgData: body.ppgData,
      timestamp: body.timestamp || new Date(),
    });

    return NextResponse.json(
      { success: true, data: newRecord },
      { status: 201 }
    );
  } catch (error: any) {
    return NextResponse.json(
      { success: false, error: error.message },
      { status: 400 }
    );
  }
}

Key Points:

  1. POST Request : This API route listens for POST requests. When data is sent to /api/save-record, it processes the data and saves it to MongoDB.

  2. JSON Format : Data is sent and received in JSON format. For example:

{
  "heartRate": { "bpm": 72, "confidence": 95 },
  "hrv": { "sdnn": 50, "confidence": 85 },
  "ppgData": [120, 125, 130, ...],
  "timestamp": "2023-10-01T12:00:00Z"
}

  1. Error Handling : If something goes wrong (e.g., invalid data), the API returns an error message with a 400 status code.

Types of HTTP Requests

Below is some common HTTP mehods supported by Next.js:

  • GET : Fetch data from the server.

  • POST : Send data to the server (used here for saving records).

  • PUT : Update existing data.

  • DELETE : Remove data.

For now, we’re focusing on the POST method to save PPG data to MongoDB.

2. How Data is Sent to MongodB

The pushDataToMongo Function

In app/page.tsx, the pushToMongoDB function sends data to the /api/save-record endpoint.

Code Walkthrough: pushDataToMongo

const pushDataToMongo = async () => {
  if (isUploading) return; // Prevent overlapping calls
  setIsUploading(true);

  if (ppgData.length === 0) {
    console.warn('No PPG data to send to MongoDB');
    return;
  }

  const recordData = {
    heartRate: { bpm: heartRate.bpm, confidence: hrv.confidence },
    hrv: { sdnn: hrv.sdnn, confidence: hrv.confidence },
    ppgData: ppgData,
    timestamp: new Date(),
  };

  try {
    const response = await fetch('/api/save-record', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(recordData),
    });

    const result = await response.json();
    if (result.success) {
      console.log('✅ Data successfully saved to MongoDB:', result.data);
    } else {
      console.error('❌ Upload failed:', result.error);
    }
  } catch (error) {
    console.error('🚨 Network error - failed to save data:', error);
  } finally {
    setIsUploading(false);
  }
};

Key Points:

  1. Fetching the API : The fetch function sends a POST request to /api/save-record.

  2. Sending Data : The recordData object contains heart rate, HRV, PPG data, and a timestamp.

  3. Handling Responses : The function logs success or failure messages based on the API response.

3. Setting Up MongoDB

Step 1: Install MongoDB dependencies

The applications work with the following dependencies:

"dependencies": {
    "@mantine/core": "^7.15.0",
    "@mantine/hooks": "^7.15.0",
    "@tensorflow/tfjs": "^4.22.0",
    "chart.js": "^4.4.7",
    "mongodb": "^6.13.0",
    "mongoose": "^8.9.6",
    "next": "15.0.4",
    "react": "^19.0.0",
    "react-chartjs-2": "^5.2.0",
    "react-dom": "^19.0.0"
  }

Installation steps:

  1. Copy the package.json file into your project directory.

  2. Run the following command to install dependencies

npm install --legacy-peer-deps

  • The --legacy-peer-deps flag ensures compatibility if students already have a Next.js project.

Step 2: Create a MongoDB Account

  1. Go to MongoDB Atlas and create an account.

  2. Create a new project and set up a free-tier cluster.

  3. Remember the password when you create the cluster, as it will be used for for configuring the database URI.

Step 3: Configure MongoDB Cluster

  • Navigate to Network Access > Add IP Address > Allow Access from Anywhere .

Step 4: Add Connection String to .env.local

  1. Create a .env.local file in your project root.

  2. Add the following line:

MONGODB_URI=mongodb+srv://<your connection string>/?retryWrites=true&w=majority&appName=Cluster0

Your connection string could be found by navigating to Clusters>Connect>MongoDB For VS Code

Testing the Set-up

  1. Start your Next.js app

npm run dev

  1. Use the "Save Data to MongoDB" button in the app to test the integration.

  2. Check your MongoDB Atlas dashboard to verify that data is being saved.
