Walkthrough of each components

1. Main Page (app/page.tsx)

The app/page.tsx file serves as the main entry point for the app. It ties all the components and hooks together, manages state, and provides buttons for recording, sampling, and saving data to MongoDB.

Key Features

  • State Management :

    • isRecording: Tracks whether the camera is recording.

    • isUploading: Prevents overlapping calls when sending data to MongoDB.

    • signalCombination: Determines how RGB values are combined into the PPG signal.

    • showConfig: Toggles configuration options.

  • Refs :

    • videoRef and canvasRef: Used to reference the video and canvas elements for capturing and processing frames.

  • Hooks :

    • usePPGProcessing: Handles frame processing, PPG signal extraction, and calculation of heart rate and HRV.

    • useSignalQuality: Evaluates the quality of the PPG signal (optional for now).

  • API Integration :

    • A button allows users to send processed data to MongoDB using the /api/save-record endpoint.

Code Explanation

const {
  ppgData,
  valleys,
  heartRate,
  hrv,
  processFrame,
  startCamera,
  stopCamera,
} = usePPGProcessing(isRecording, signalCombination, videoRef, canvasRef);

  • The usePPGProcessing hook processes video frames and extracts PPG data, valleys, heart rate, and HRV.

  • processFrame is called repeatedly during recording to analyze each frame.

  • startCamera and stopCamera manage the webcam feed.

2. Components

2.1 CameraFeed

The CameraFeed component displays the webcam feed and provides a canvas for processing video frames.

Purpose

  • Captures video frames from the webcam.

  • Displays the live feed on the screen.

interface CameraFeedProps {
  videoRef: React.RefObject<HTMLVideoElement>;
  canvasRef: React.RefObject<HTMLCanvasElement>;
}

  return (
    <div>
      {/* Video Element */}

      <canvas
        ref={canvasRef}
        width={640}
        height={480}
        className="w-full max-w-[640px] h-auto border border-black"
      />
    </div>
  );
};

  • <video>: Hidden live webcam feed.

  • <canvas>: Draw video frame to be visualized and processed

2.2 ChartComponent

The ChartComponent visualizes the PPG signal and detected valleys using react-chartjs-2.

Purpose

  • Displays the PPG signal as a smooth line graph.

  • Highlights detected valleys (heartbeats) as red dots.

export default function ChartComponent({ ppgData, valleys }: ChartComponentProps) {
  const chartData = {
    labels: Array.from({ length: ppgData.length }, (_, i) => i.toString()),
    datasets: [
      {
        label: 'PPG Signal',
        data: ppgData,
        borderColor: 'rgb(75, 192, 192)',
        tension: 0.4,
        fill: true,
        backgroundColor: 'rgba(75, 192, 192, 0.2)',
        pointRadius: 0,
      },
      {
        label: 'Valleys',
        data: ppgData.map((_, i) => valleys.find((v) => v.index === i)?.value || null),
        pointBackgroundColor: 'red',
        pointRadius: 3,
        showLine: false,
      },
    ],
  };

  const chartOptions = {
    responsive: true,
    scales: {
      y: {
        beginAtZero: false,
      },
    },
    animation: {
      duration: 0, // Disable animation for better performance
    },
  };

  return <Line data={chartData} options={chartOptions} />;
}

  • ppgData: Array of PPG signal values.

  • valleys: Detected valleys (heartbeats) displayed as red dots.

  • chartOptions: Configures the chart to be responsive and disables animations for performance.

2.3 MetricsCard

The MetricsCard component displays metrics like heart rate, HRV, and confidence levels.

Purpose

  • Shows calculated metrics in a clean, user-friendly format.

  • Handles fallback values (--) for undefined or invalid data.

export default function MetricsCard({ title, value, unit, confidence }: MetricsCardProps) {
  return (
    <div>
      <h3>{title}</h3>
      <p>
        {typeof value === 'number' && value > 0
          ? `${value} ${unit || ''}`
          : typeof value === 'object' && value !== null
          ? value.bpm !== undefined
            ? `${value.bpm} BPM`
            : value.sdnn !== undefined
            ? isNaN(value.sdnn)
              ? '--'
              : `${value.sdnn} ms`
            : '--'
          : '--'}
      </p>
      {confidence !== undefined && <p>Confidence: {confidence.toFixed(1)}%</p>}
    </div>
  );
}

2.4 SignalCombinationSelector

The SignalCombinationSelector allows users to choose how RGB values are combined into the PPG signal.

Purpose

  • Provides options like "Red Only," "Green Only," and "Default (2R - G - B)."

  • Updates the signalCombination state in the parent component.

export default function SignalCombinationSelector({
  signalCombination,
  setSignalCombination,
}: SignalCombinationSelectorProps) {
  const options = [
    { value: 'default', label: 'Default (2R - G - B)' },
    { value: 'redOnly', label: 'Red Only' },
    { value: 'greenOnly', label: 'Green Only' },
    { value: 'blueOnly', label: 'Blue Only' },
    { value: 'redMinusBlue', label: 'Red - Blue' },
    { value: 'custom', label: 'Custom (3R - G - B)' },
  ];

  return (
    <select
      value={signalCombination}
      onChange={(e) => setSignalCombination(e.target.value)}
      className="mt-1 block w-full pl-3 pr-10 py-2 text-base border-gray-300 focus:outline-none focus:ring-cyan-500 focus:border-cyan-500 sm:text-sm rounded-md"
    >
      {options.map((option) => (
        <option key={option.value} value={option.value}>
          {option.label}
        </option>
      ))}
    </select>
  );
}

  • A dropdown menu with predefined options.

  • Updates the signalCombination state when the user selects a new option.
