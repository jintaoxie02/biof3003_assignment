Implementing GET Requests and Refactoring MongoDB Logic

By the end of this session, students will:

  1. Understand how to make GET requests in React to retrieve data from MongoDB.

  2. Learn to refactor repetitive code into custom hooks for better readability and maintainability.

  3. Calculate and display historical averages (heart rate and HRV) from stored MongoDB records.

  4. Practice integrating backend APIs with frontend components.

1. Why Refactor MongoDB Logic into a Hook?

In the previous session, the pushDataToMongo functions were defined directly in page.tsx. While this works, it makes the code harder to maintain.

By moving these functions into a custom hook (useMongoDB), we:

  • Keep page.tsx focused on UI and state management.

  • Encapsulate database logic for reuse across components.

  • Simplify debugging and testing.

2. The useMongoDB Hook: Code Breakdown

This hook handles POST (saving data) and GET (fetching historical averages) operations with MongoDB.

Code for hooks/useMongoDB.ts

// hooks/useMongoDB.ts
import { useState } from 'react';

interface HistoricalData {
  avgHeartRate: number;
  avgHRV: number;
}

export default function useMongoDB() {
  const [isUploading, setIsUploading] = useState(false);
  const [historicalData, setHistoricalData] = useState<HistoricalData>({
    avgHeartRate: 0,
    avgHRV: 0,
  });

  // POST: Save data to MongoDB
  const pushDataToMongo = async (recordData: RecordData) => {
    if (isUploading) return; // Prevent overlapping calls
    setIsUploading(true);
    try {
      const response = await fetch('/api/handle-record', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(recordData),
      });
      const result = await response.json();
      if (result.success) {
        console.log('✅ Data saved:', result.data);
      } else {
        console.error('❌ Error:', result.error);
      }
    } catch (error) {
      console.error('🚨 Network error:', error);
    } finally {
      setIsUploading(false);
    }
  };

  // GET: Fetch historical averages
  const fetchHistoricalData = async () => {
    try {
      const response = await fetch('/api/handle-record', {
        method: 'GET',
        headers: { 'Content-Type': 'application/json' },
      });
      const result = await response.json();
      if (result.success) {
        setHistoricalData({
          avgHeartRate: result.avgHeartRate,
          avgHRV: result.avgHRV,
        });
      } else {
        console.error('❌ Error:', result.error);
      }
    } catch (error) {
      console.error('🚨 Network error:', error);
    }
  };

  return {
    isUploading,
    pushDataToMongo,
    fetchHistoricalData,
    historicalData,
  };
}

Key Features

  1. State Management :

    • isUploading: Tracks whether data is being uploaded.

    • historicalData: Stores the fetched averages (HR and HRV).

  2. POST Function :

    • pushDataToMongo: Sends data to the /api/handle-record endpoint.

  3. GET Function :

    • fetchHistoricalData: Retrieves historical averages from the same endpoint.

3. Updated API Route: /api/handle-record/route.ts

This single API route handles both POST and GET requests for MongoDB operations.

// app/api/handle-record/route.ts
import { NextResponse } from 'next/server';
import mongoose from 'mongoose';

const MONGODB_URI = process.env.MONGODB_URI as string;
if (!MONGODB_URI) {
  throw new Error('MONGODB_URI not defined');
}

let cached = (global as any).mongoose;
if (!cached) {
  cached = (global as any).mongoose = { conn: null, promise: null };
}

async function dbConnect() {
  if (cached.conn) return cached.conn;
  if (!cached.promise) {
    cached.promise = mongoose.connect(MONGODB_URI, { bufferCommands: false });
  }
  cached.conn = await cached.promise;
  return cached.conn;
}

const RecordSchema = new mongoose.Schema({
  heartRate: { bpm: Number, confidence: Number },
  hrv: { sdnn: Number, confidence: Number },
  ppgData: [Number],
  timestamp: { type: Date, default: Date.now },
});

const Record = mongoose.models.Record || mongoose.model('Record', RecordSchema);

// POST Handler
export async function POST(request: Request) {
  try {
    await dbConnect();
    const body = await request.json();
    const newRecord = await Record.create(body);
    return NextResponse.json({ success: true, data: newRecord }, { status: 201 });
  } catch (error: any) {
    return NextResponse.json({ success: false, error: error.message }, { status: 400 });
  }
}

// GET Handler
export async function GET() {
  try {
    await dbConnect();
    const pipeline = [
      {
        $group: {
          _id: null,
          avgHeartRate: { $avg: '$heartRate.bpm' },
          avgHRV: { $avg: '$hrv.sdnn' },
        },
      },
    ];
    const result = await Record.aggregate(pipeline);
    return NextResponse.json({ success: true, ...result[0] }, { status: 200 });
  } catch (error: any) {
    return NextResponse.json({ success: false, error: error.message }, { status: 500 });
  }
}

4. Understanding the API Route

The API route (/app/api/handle-record/route.ts) handles two types of HTTP requests: POST and GET. These requests allow the application to interact with the MongoDB database:

  1. POST Request : Used to save new data (e.g., heart rate, HRV, PPG data) to the database.

  2. GET Request : Used to fetch historical data (e.g., average heart rate and HRV) from the database.

Let’s dive into how each request works in detail.

4.1 POST Request: Saving Data to MongoDB

The POST request allows the client (your React app) to send new data to the server and store it in the MongoDB database. For example, when the user clicks the "Save Data to MongoDB" button, the app sends the recorded heart rate, HRV, and PPG data to this API endpoint.

Here’s a breakdown of the POST handler:

export async function POST(request: Request) {
  try {
    // Step 1: Connect to the database
    await dbConnect();

    // Step 2: Parse the incoming JSON data from the request body
    const body = await request.json();

    // Step 3: Create a new record in the database using the parsed data
    const newRecord = await Record.create(body);

    // Step 4: Respond with success and the newly created record
    return NextResponse.json({ success: true, data: newRecord }, { status: 201 });
  } catch (error: any) {
    // Step 5: Handle errors and respond with an error message
    return NextResponse.json({ success: false, error: error.message }, { status: 400 });
  }
}

Step-by-Step Explanation

  1. Database Connection :

    • The dbConnect() function ensures that the server is connected to the MongoDB database before performing any operations.

    • If the connection already exists (cached), it reuses it to avoid redundant connections.

  2. Parsing the Request Body :

    • The request.json() method extracts the data sent by the client (React app) as JSON.

    • This data includes fields like heartRate, hrv, ppgData, and timestamp.

  3. Creating a New Record :

    • The Record.create(body) method inserts the parsed data into the records collection in MongoDB.

    • Mongoose automatically validates the data against the schema (RecordSchema) to ensure all required fields are present and correctly formatted.

  4. Responding to the Client :

    • If the operation succeeds, the server responds with a 201 Created status and the newly created record.

    • If there’s an error (e.g., invalid data or database issues), the server responds with a 400 Bad Request status and an error message.

4.2 GET Request: Fetching Historical Data

The GET request retrieves historical data from the database. Specifically, it calculates the average heart rate (avgHeartRate) and average HRV (avgHRV) across all records in the database.

Here’s a breakdown of the GET handler:

export async function GET() {
  try {
    // Step 1: Connect to the database
    await dbConnect();

    // Step 2: Define an aggregation pipeline to calculate averages
    const pipeline = [
      {
        $group: {
          _id: null,
          avgHeartRate: { $avg: '$heartRate.bpm' },
          avgHRV: { $avg: '$hrv.sdnn' },
        },
      },
    ];

    // Step 3: Execute the aggregation query
    const result = await Record.aggregate(pipeline);

    // Step 4: Respond with success and the calculated averages
    return NextResponse.json({ success: true, ...result[0] }, { status: 200 });
  } catch (error: any) {
    // Step 5: Handle errors and respond with an error message
    return NextResponse.json({ success: false, error: error.message }, { status: 500 });
  }
}

Step-by-Step Explanation

  1. Database Connection :

    • Similar to the POST request, the dbConnect() function ensures the server is connected to MongoDB.

  2. Aggregation Pipeline :

    • The $group stage groups all records together (using _id: null) and calculates the following:

      • avgHeartRate: The average of the bpm field in the heartRate object.

      • avgHRV: The average of the sdnn field in the hrv object.

    • Aggregation pipelines are powerful tools in MongoDB for performing complex calculations on large datasets.

  3. Executing the Query :

    • The Record.aggregate(pipeline) method runs the aggregation query and returns the results.

    • The result is an array where each element represents a group (in this case, just one group since _id: null).

  4. Responding to the Client :

    • If the operation succeeds, the server responds with a 200 OK status and the calculated averages (avgHeartRate and avgHRV).

    • If there’s an error (e.g., database issues), the server responds with a 500 Internal Server Error status and an error message.

4.3 How the Hooks Use These Requests

In your React app, the useMongoDB hook interacts with the API to handle POST and GET requests. Let’s see how it works:

Pushing Data to MongoDB (POST)

The pushDataToMongo function sends a POST request to the /api/handle-record endpoint:

const pushDataToMongo = async (recordData: RecordData) => {
  if (isUploading) return; // Prevent overlapping calls

  setIsUploading(true); // Lock the function
  try {
    const response = await fetch('/api/handle-record', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(recordData),
    });

    const result = await response.json();
    if (result.success) {
      console.log('✅ Data successfully saved to MongoDB:', result.data);
    } else {
      console.error('❌ Upload failed:', result.error);
    }
  } catch (error) {
    console.error('🚨 Network error - failed to save data:', error);
  } finally {
    setIsUploading(false); // Unlock the function
  }
};

  • The fetch function sends a POST request with the recordData (e.g., heart rate, HRV, PPG data) as the request body.

  • The server processes the request and saves the data to MongoDB.

Fetching Historical Data (GET)

The fetchHistoricalData function sends a GET request to the same endpoint:

const fetchHistoricalData = async () => {
  try {
    const response = await fetch('/api/handle-record', {
      method: 'GET',
      headers: { 'Content-Type': 'application/json' },
    });

    const result = await response.json();
    if (result.success) {
      setHistoricalData({
        avgHeartRate: result.avgHeartRate,
        avgHRV: result.avgHRV,
      });
    } else {
      console.error('❌ Failed to fetch historical data:', result.error);
    }
  } catch (error) {
    console.error('🚨 Network error - failed to fetch historical data:', error);
  }
};

  • The fetch function sends a GET request to retrieve the historical averages (avgHeartRate and avgHRV).

  • The server processes the request and returns the calculated averages.

4.4 Summary

  • POST Request :

    • Sends new data (e.g., heart rate, HRV, PPG data) to the server.

    • The server saves the data to MongoDB and responds with the newly created record.

  • GET Request :

    • Retrieves historical data (e.g., average heart rate and HRV) from the server.

    • The server calculates the averages using an aggregation pipeline and responds with the results.
